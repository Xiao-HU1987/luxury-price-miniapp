## 1. 产品概述

### 1.1 产品定位

全球奢侈品价格比价小程序，帮助用户快速查询全球各地奢侈品价格，找到最优惠的购买渠道。

### 1.2 产品架构

```
┌─────────────────────────────────────┐
│         微信小程序（用户端）          │
│  比价 / 优惠 / 汇率 / 买手 / 我的     │
└────────────────┬────────────────────┘
                 │ HTTPS
┌────────────────▼────────────────────┐
│          FastAPI 后端服务            │
│  商品/用户/订单/支付/营销/统计        │
└────────────────┬────────────────────┘
                 │
┌────────────────▼────────────────────┐
│          MySQL / SQLite 数据库       │
└─────────────────────────────────────┘
                 ▲
                 │
┌────────────────┴────────────────────┐
│       Vue3 + Element Plus 后台       │
│  商品/用户/订单/营销/数据/系统        │
└─────────────────────────────────────┘
```

### 2.8 订单流程

```
创建订单 → 待支付 → 支付成功 → 待发货 → 已发货 → 待收货 → 确认收货 → 已完成
               ↓                        ↓
            取消订单                  申请退款
```

**订单状态说明**：

| 状态 | 说明 | 可执行操作 |
|------|------|-----------|
| pending 待支付 | 订单已创建，等待付款 | 去支付、取消订单 |
| paid 待发货 | 支付成功，等待商家发货 | 查看详情 |
| shipped 待收货 | 已发货，运输中 | 查看物流、确认收货 |
| completed 已完成 | 交易完成 | 查看详情 |
| cancelled 已取消 | 订单已取消 | 查看详情 |

## 3. 用户角色与权限

### 3.1 角色定义

| 角色 | 说明 | 权限范围 |
|------|------|----------|
| 普通用户 | 微信登录的普通用户 | 浏览商品、收藏、下单、发布需求 |
| VIP 用户 | 购买了 VIP 套餐的用户 | 普通用户权限 + VIP 专属返点 + 优先匹配 |
| 买手用户 | 认证为买手的用户 | 接单、管理订单 |
| 管理员 | 后台管理人员 | 全部后台功能 |
| 超级管理员 | 最高权限 | 所有权限 + 管理员管理 |

### 3.2 用户状态

| 状态 | 说明 |
|------|------|
| active 正常 | 正常使用 |
| disabled 禁用 | 账号被禁用，无法登录 |

### 3.3 用户注册与登录

- **登录方式**：微信一键登录（无需注册）
- **首次登录**：自动创建用户账号，生成唯一 user_id
- **用户信息**：头像、昵称（从微信授权获取）
- **手机号**：用户可在个人中心手动绑定

## 4. 核心业务逻辑

### 4.1 商品价格体系

```
SPU（标准产品单元）
  ├── SKU1（规格1，如：中号/黑色）
  │     ├── 中国价格
  │     ├── 日本价格
  │     ├── 法国价格
  │     └── ...
  ├── SKU2（规格2，如：小号/棕色）
  │     ├── 中国价格
  │     └── ...
  └── ...
```

- **SPU**：商品本身，如「Chanel Classic Flap」
- **SKU**：具体规格，如「中号/黑色/金扣」
- **SKUPrice**：某 SKU 在某国家/地区的价格

### 4.2 汇率换算逻辑

- 基准货币：人民币（CNY）
- 汇率数据：后台维护，管理员可手动更新
- 前端换算：`外币价格 / 汇率 = 人民币价格`
- 价格展示：保留整数，千分位格式化

### 4.3 开屏广告展示逻辑

1. 小程序启动 → 请求开屏广告接口
2. 后台返回当前启用的广告
3. 前端检查「今日是否已展示过」
   - 已达每日上限 → 直接跳首页
   - 未达上限 → 展示广告，本地记录展示次数
4. 倒计时结束 / 用户点击跳过 → 跳转首页
5. 展示/点击行为上报后台，用于数据统计

### 4.4 订单支付流程

```
用户下单
    ↓
后端创建订单（状态：pending）
    ↓
调用微信支付统一下单
    ↓
前端调起微信支付（wx.requestPayment）
    ↓
用户支付成功 → 微信回调后端
    ↓
后端更新订单状态（paid）+ 记录支付时间
    ↓
前端查询订单状态 → 展示支付结果
```

> 调试模式下支持「模拟支付」功能，无需真实微信支付即可测试完整流程。

### 4.5 VIP 会员逻辑

- 购买 VIP 套餐 → 创建 VIP 订单
- 支付成功 → 更新用户 is_vip = true，设置 vip_expire_time
- 到期自动失效（可在后台手动延长）
- 前端判断 VIP 状态：读取 userInfo.is_vip 字段

### 5.3 商品管理

**入口**：左侧菜单 → 商品管理

**功能模块**：

#### 品牌管理
- 品牌列表：品牌 ID、名称（中/英）、Logo、国家
- 新增品牌：填写品牌信息
- 编辑品牌：修改品牌信息
- 删除品牌：删除品牌（注意：删除前确保没有商品使用该品牌）

#### 分类管理
- 分类列表：分类 ID、名称、排序
- 新增/编辑/删除分类

#### SPU 管理
- 商品 SPU 列表
- 新增 SPU：选择品牌、分类，填写名称（中/英）、货号、描述、图片等
- 编辑 SPU：修改 SPU 信息
- 删除 SPU：删除时会级联删除关联的 SKU 和价格

#### SKU 管理
- 查看某个 SPU 下的所有 SKU
- 新增 SKU：规格名称、规格值、图片
- 编辑/删除 SKU

#### SKU 价格管理
- 管理每个 SKU 在不同国家/地区的价格
- 新增价格：选择国家、货币、价格、库存、门店
- 编辑价格：修改价格和库存
- 删除价格

#### 商品搜索
- 按关键词、品牌、分类搜索商品
- 查看商品详情

> **操作建议**：先建品牌和分类 → 再建 SPU → 再建 SKU → 最后添加各地区价格。

### 5.5 订单管理

**入口**：左侧菜单 → 订单管理

**功能**：

- 订单详情：查看订单完整信息

  - 待支付 → 已支付（线下收款时手动标记）

**订单状态流转**：
```
待支付(pending) → 待发货(paid) → 待收货(shipped) → 已完成(completed)
     ↓                ↓                ↓
  已取消(cancelled)   ↓                ↓
                      ↓                ↓
                   已退款(refunded)    ↓
                                       ↓
                                    已退款(refunded)
```

## 6. 常见问题 FAQ

### 6.1 小程序端常见问题

**Q：为什么看不到开屏广告？**
A：请检查：1）后台广告是否启用；2）今日是否已展示过（每日上限）；3）网络是否正常。

**Q：为什么商品价格不对？**
A：价格为后台手动维护，可能存在滞后。如有疑问请联系管理员更新。

**Q：汇率多久更新一次？**
A：汇率由管理员手动更新，建议每日更新一次。

**Q：如何开通 VIP？**
A：进入「我的」→「VIP 会员」→ 选择套餐 → 微信支付开通。

---

### 6.2 后台管理常见问题

**Q：忘记管理员密码怎么办？**
A：联系技术人员在后端配置中重置密码。

**Q：如何批量导入商品？**
A：当前版本暂不支持批量导入，需手动逐条添加。后续版本可开发 Excel 导入功能。

**Q：订单支付失败怎么办？**
A：检查：1）微信支付商户号配置是否正确；2）支付证书是否过期；3）回调地址是否公网可访问。

**Q：如何给用户手动开通 VIP？**
A：用户管理 → 找到对应用户 → 编辑 → 打开「VIP」开关 → 保存。

**Q：开屏广告可以同时展示多个吗？**
A：同一时间只会展示一个启用中的广告。建议每次只启用一个。

**Q：用户被禁用后还能登录吗？**
A：不能。禁用后用户登录会提示账号异常。
