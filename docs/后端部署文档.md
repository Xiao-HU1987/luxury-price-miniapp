# 后端部署文档

## 一、环境准备

### 1. 安装 Python 3.8+

```bash
# Ubuntu/Debian
sudo apt update
sudo apt install python3 python3-pip

# CentOS
sudo yum install python3 python3-pip
```

### 2. 安装 MySQL

```bash
# Ubuntu/Debian
sudo apt install mysql-server

# 创建数据库
mysql -u root -p
CREATE DATABASE luxury_price CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE USER 'luxury'@'localhost' IDENTIFIED BY 'your_password';
GRANT ALL PRIVILEGES ON luxury_price.* TO 'luxury'@'localhost';
FLUSH PRIVILEGES;
```

### 3. 安装依赖

```bash
cd /path/to/server
pip3 install -r requirements.txt
```

## 二、配置环境变量

```bash
# 复制配置模板
cp .env.example .env

# 编辑配置文件
nano .env
```

**必须修改的配置**：
- `SECRET_KEY`：更换为随机字符串
- `DATABASE_URL`：填写真实数据库连接
- `WECHAT_APPID`：填写小程序 AppID
- `WECHAT_SECRET`：填写小程序 AppSecret
- `DEBUG=False`：关闭调试模式
- `TRINO_ENABLED=True`：启用 Trino 读接口
- `TRINO_HOST=localhost`、`TRINO_PORT=8080`：Trino 服务地址
- `TRINO_USER`、`TRINO_PASSWORD`：Trino 连接账号
- `TRINO_CATALOG`、`TRINO_SCHEMA`：Trino 数据源位置

## 三、启动后端服务

如果你启用了 Trino 读层，请先确认 `localhost:8080` 已有可用 Trino 服务，并且后端 `.env` 中的 catalog/schema 与实际表结构一致。

### 方式一：直接启动（开发/测试）

```bash
cd /path/to/server
python3 app.py
```

### 方式二：uvicorn 启动（推荐）

```bash
cd /path/to/server
uvicorn app:app --host 0.0.0.0 --port 8000
```

### 方式三：后台运行

```bash
nohup uvicorn app:app --host 0.0.0.0 --port 8000 > server.log 2>&1 &
```

## 四、Nginx 配置

### 1. 安装 Nginx

```bash
sudo apt install nginx
```

### 2. 配置 SSL 证书

将 SSL 证书文件放到 `/etc/nginx/ssl/` 目录：
- `domain.crt`：证书文件
- `domain.key`：私钥文件

### 3. 创建 Nginx 配置

```bash
sudo nano /etc/nginx/sites-available/luxury-price
```

配置内容：

```nginx
server {
    listen 443 ssl;
    server_name 你的域名.com;

    # SSL 证书配置
    ssl_certificate /etc/nginx/ssl/domain.crt;
    ssl_certificate_key /etc/nginx/ssl/domain.key;
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers HIGH:!aNULL:!MD5;

    # 后端 API 代理
    location /api/ {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    # 后台管理前端（Vue 构建）
    location /admin/ {
        alias /path/to/admin/dist/;
        try_files $uri $uri/ /index.html;
    }

    # 根路径
    location / {
        return 200 '{"status": "ok", "message": "奢侈品比价 API"}';
    }
}

# HTTP 重定向到 HTTPS
server {
    listen 80;
    server_name 你的域名.com;
    return 301 https://$server_name$request_uri;
}
```

### 4. 启用配置

```bash
sudo ln -s /etc/nginx/sites-available/luxury-price /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl restart nginx
```

## 五、使用 Systemd 管理服务

创建服务配置文件：

```bash
sudo nano /etc/systemd/system/luxury-api.service
```

内容：

```ini
[Unit]
Description=Luxury Price API
After=network.target

[Service]
User=www-data
Group=www-data
WorkingDirectory=/path/to/server
Environment="PATH=/path/to/server"
ExecStart=/usr/bin/python3 -m uvicorn app:app --host 127.0.0.1 --port 8000
Restart=always
RestartSec=3

[Install]
WantedBy=multi-user.target
```

启动服务：

```bash
sudo systemctl daemon-reload
sudo systemctl start luxury-api
sudo systemctl enable luxury-api  # 开机自启
sudo systemctl status luxury-api  # 查看状态
```

## 八、常见问题

### Q：502 Bad Gateway
检查：1）后端服务是否启动；2）端口是否正确；3）防火墙是否开放。

### Q：SSL 证书无效
检查：1）证书路径是否正确；2）证书是否过期；3）域名是否匹配。

### Q：数据库连接失败
检查：1）DATABASE_URL 格式是否正确；2）MySQL 用户权限；3）数据库是否创建。

### Q：Trino 读接口没有数据
检查：1）`TRINO_ENABLED` 是否为 `True`；2）Trino 服务是否监听在 `localhost:8080`；3）`TRINO_CATALOG` 和 `TRINO_SCHEMA` 是否与表结构一致；4）后端运行账号是否有对应表的读取权限。
